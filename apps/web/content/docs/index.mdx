Welcome. **Offerkit** is an MIT-licensed alternative to Voucherify you run yourself. One published Docker image, one Docker compose file, or two Railway Docker Image services, and you have:

- a **dashboard** to manage campaigns, vouchers, customers, segments, loyalty, referrals, webhooks, API keys
- a **REST API** at `/api/v1/...` with the same shape you'd hit through the typed SDK
- a typed **TypeScript SDK** — no codegen, types flow from the oRPC contract
- a **CLI** (`offerkit`) with verbs for the most common back-office tasks
- a curated **MCP server** so an agent can drive your promotion stack

## Domain model at a glance

A **campaign** is a parent for one of five kinds of programs (`DISCOUNT`, `GIFT_VOUCHERS`, `LOYALTY_PROGRAM`, `REFERRAL_PROGRAM`, `PROMOTION`). A **voucher** belongs to a discount or gift campaign and either carries a discount config or a gift-card balance. A **loyalty program** has tiers, earning rules, rewards, and member ledgers. A **referral program** issues `{prefix}-{code}` codes that, on convert, hand out two configurable rewards. **Validation rules** live in a reusable JSON-Logic library. Every state-changing operation emits an **event**; **webhooks** subscribe to event types and receive HMAC-signed POSTs with exponential-backoff retries.
