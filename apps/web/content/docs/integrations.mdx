---
title: SDK, CLI, and MCP
description: Three programmatic surfaces over the same typed contract.
---

All three call the same `/api/v1/...` REST API and authenticate with the same `OFFERKIT_API_KEY`.

## TypeScript SDK

```bash
pnpm add @offerkit/sdk
```

```ts
import { createClient, verifyWebhook } from "@offerkit/sdk";

const offerkit = createClient({
  baseUrl: "https://your-offerkit-deployment",
  apiKey: process.env.OFFERKIT_API_KEY!,
});

const result = await offerkit.vouchers.redeem({
  code: "SUMMER10",
  order: { amount: 9999, currency: "USD", items: [] },
  idempotencyKey: "order-42",
});
```

Types flow directly from the oRPC contract — no codegen step. Edge-runtime safe (`node:crypto` is only used for the optional `verifyWebhook` helper).

## CLI (`offerkit`)

```bash
pnpm add -g @offerkit/cli
offerkit login --url https://your-offerkit-deployment --api-key offerkit_…

offerkit vouchers list
offerkit vouchers validate SUMMER10 --amount 9999
offerkit vouchers redeem SUMMER10 --amount 9999 --idempotency-key order-42

offerkit campaigns create --name "Spring 2026" --type DISCOUNT --currency USD
offerkit customers list --search alice
```

Config is stored in `~/.offerkitrc` (chmod 0600). `OFFERKIT_API_URL` / `OFFERKIT_API_KEY` env vars override the file.

## MCP server

`offerkit-mcp` is a stdio MCP server exposing 11 curated tools (the curation is intentional — destructive verbs are explicitly listed so an agent can't accidentally invoke procedures we'd rather it not). Mutating tools (`vouchers_redeem`, `vouchers_stack_redeem`) carry "confirm with the user" guidance in their description.

Add to your MCP host (Claude Desktop, Cursor, …):

```json
{
  "mcpServers": {
    "offerkit": {
      "command": "npx",
      "args": ["-y", "@offerkit/mcp"],
      "env": {
        "OFFERKIT_API_URL": "https://your-offerkit-deployment",
        "OFFERKIT_API_KEY": "offerkit_…"
      }
    }
  }
}
```

Available tools: `vouchers_list`, `vouchers_get`, `vouchers_validate`, `vouchers_redeem`, `vouchers_stack_redeem`, `campaigns_list`, `campaigns_get`, `customers_list`, `customers_get`, `loyalty_member_history`, `segments_preview`.
