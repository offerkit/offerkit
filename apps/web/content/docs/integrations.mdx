---
title: SDK, CLI, and MCP
description: Three programmatic surfaces over the same typed contract.
---

All three call the same `/api/v1/...` REST API and authenticate with the same `OVX_API_KEY`.

## TypeScript SDK

```bash
pnpm add @open-voucherify/sdk
```

```ts
import { createClient, verifyWebhook } from "@open-voucherify/sdk";

const ovx = createClient({
  baseUrl: "https://your-ovx-deployment",
  apiKey: process.env.OVX_API_KEY!,
});

const result = await ovx.vouchers.redeem({
  code: "SUMMER10",
  order: { amount: 9999, currency: "USD", items: [] },
  idempotencyKey: "order-42",
});
```

Types flow directly from the oRPC contract — no codegen step. Edge-runtime safe (`node:crypto` is only used for the optional `verifyWebhook` helper).

## CLI (`ovx`)

```bash
pnpm add -g @open-voucherify/cli
ovx login --url https://your-ovx-deployment --api-key ovx_…

ovx vouchers list
ovx vouchers validate SUMMER10 --amount 9999
ovx vouchers redeem SUMMER10 --amount 9999 --idempotency-key order-42

ovx campaigns create --name "Spring 2026" --type DISCOUNT --currency USD
ovx customers list --search alice
```

Config is stored in `~/.ovxrc` (chmod 0600). `OVX_API_URL` / `OVX_API_KEY` env vars override the file.

## MCP server

`ovx-mcp` is a stdio MCP server exposing 11 curated tools (the curation is intentional — destructive verbs are explicitly listed so an agent can't accidentally invoke procedures we'd rather it not). Mutating tools (`vouchers_redeem`, `vouchers_stack_redeem`) carry "confirm with the user" guidance in their description.

Add to your MCP host (Claude Desktop, Cursor, …):

```json
{
  "mcpServers": {
    "open-voucherify": {
      "command": "npx",
      "args": ["-y", "@open-voucherify/mcp"],
      "env": {
        "OVX_API_URL": "https://your-ovx-deployment",
        "OVX_API_KEY": "ovx_…"
      }
    }
  }
}
```

Available tools: `vouchers_list`, `vouchers_get`, `vouchers_validate`, `vouchers_redeem`, `vouchers_stack_redeem`, `campaigns_list`, `campaigns_get`, `customers_list`, `customers_get`, `loyalty_member_history`, `segments_preview`.
