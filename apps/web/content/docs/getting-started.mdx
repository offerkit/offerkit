## 1. Run the stack

```bash
git clone https://github.com/offerkit/offerkit
cd offerkit
cp .env.example .env  # set ADMIN_EMAIL, ADMIN_PASSWORD, BETTER_AUTH_SECRET
docker compose up -d
```

Compose pulls the published `ghcr.io/offerkit/offerkit` image and runs it as both `web` and `worker`. The web app is at `http://localhost:3000`, the worker exposes a health endpoint at `:9091`, Postgres listens on `:5432`. First boot creates an admin from `ADMIN_EMAIL` / `ADMIN_PASSWORD` and flags `mustChangePassword`.

## 4. End-to-end redemption

```bash
# Create a discount campaign
curl -H "Authorization: Bearer $OFFERKIT_API_KEY" -H 'Content-Type: application/json' \
  -X POST $OFFERKIT_API_URL/api/v1/campaigns \
  -d '{"name":"Spring 2026","type":"DISCOUNT","currency":"USD"}'

# Create a 1000-cent voucher inside it (use the campaignId from above)
curl -H "Authorization: Bearer $OFFERKIT_API_KEY" -H 'Content-Type: application/json' \
  -X POST $OFFERKIT_API_URL/api/v1/vouchers \
  -d '{"campaignId":"â€¦","type":"DISCOUNT","discount":{"type":"AMOUNT","amount":1000}}'

# Validate (read-only preview)
curl -H "Authorization: Bearer $OFFERKIT_API_KEY" -H 'Content-Type: application/json' \
  -X POST $OFFERKIT_API_URL/api/v1/vouchers/<CODE>/validate \
  -d '{"order":{"amount":5000,"currency":"USD","items":[]}}'

# Redeem (commits a row)
curl -H "Authorization: Bearer $OFFERKIT_API_KEY" -H 'Content-Type: application/json' \
  -X POST $OFFERKIT_API_URL/api/v1/vouchers/<CODE>/redemption \
  -d '{"order":{"amount":5000,"currency":"USD","items":[]},"idempotencyKey":"order-123"}'
```

The redeem endpoint is idempotent on `(voucherCode, idempotencyKey)` â€” replays return the same response with `idempotent: true`. Validate, redeem, and stack-redeem failures also include an `explanations` array with support-safe reason codes and details such as activation windows, redemption limits, gift card balance, and stacking skips.
